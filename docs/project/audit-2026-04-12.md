## КРИТИЧНІ БАГИ (ламають функціонал)

### ✅ A1. Альбоми: 3 відповіді замість 1 — ВИПРАВЛЕНО 2026-04-12
**Файл:** `app/message_logic.py:924`
**Суть:** Гейт альбомів `msg.media_group_id and _should_use_media_route(geometry)` вимагає `geometry.addressed == True`. Але в альбомі з 3 фото тільки ОДНЕ (з caption/@mention) має `addressed=True`. Решта 2 items обходять альбомний шлях і потрапляють у звичайний текстовий пайплайн — кожне провокує окрему відповідь бота.
**Фікс:** Якщо `msg.media_group_id` є — або claim-and-process, або `return` (тиха відсічка). Ніколи не пускати album member у звичайний пайплайн.

### ✅ A2. Альбоми: бот не бачить альбом взагалі — ВИПРАВЛЕНО 2026-04-12
**Файл:** `app/message_logic.py:924-940`
**Суть:** Якщо перше повідомлення альбому (яке виграє claim) — це фото БЕЗ caption, а caption є на другому/третьому — переможець має порожній `user_text`, `build_user_task` повертає `None`, бот мовчить.
**Фікс:** Caption/текст треба брати з усього альбому (об'єднувати captions всіх items).

### ✅ A3. `runner.py:1319` — `fetch_page` не імпортовано — ВИПРАВЛЕНО 2026-04-12
**Файл:** `agent/runner.py:1319`
**Суть:** Функція `fetch_page` використовується в tool loop, але ніде не імпортована. `NameError` при кожному виклику tool `fetch_page`.
**Фікс:** Додати імпорт `from agent.tools.fetch_page import fetch_page`.

### ✅ A4. `runner.py:1341` — agent tool loop працює тільки 1 крок — ВИПРАВЛЕНО 2026-04-12
**Файл:** `agent/runner.py:1341`
**Суть:** Після обробки tool results, наступний `chat_once` викликається з `tools=None`. Модель не може зробити другий tool call — loop завжди виходить після 1 ітерації. `max_steps` де-факто ігнорується.
**Фікс:** Передавати `tools=tools` замість `tools=None`.

### A5. Telethon: бот не бачить @mention в групових чатах
**Файл:** `app/chat_geometry.py:368-369`
**Суть:** `_has_mention_text()` перевіряє ВЖЕ ОЧИЩЕНИЙ текст (після `_strip_bot_mention`), тому `@bot` вже видалено. `_has_mention_text("hello", "bot")` завжди `False`. В групових чатах через Telethon бот не вважає себе addressed при @mention.
**Фікс:** Перевіряти mention на ОРИГІНАЛЬНОМУ тексті (`event.raw_text`), а не на `clean_text`.

### ✅ A6. `.content.strip()` без None-guard — crash — ВИПРАВЛЕНО 2026-04-12
**Файли:** `memory/importance.py:76`, `memory/reflection.py:94`
**Суть:** `resp.choices[0].message.content.strip()` впаде з `AttributeError` якщо LLM поверне `content=None`. Решта кодової бази використовує `(... .content or "").strip()`.
**Фікс:** Додати `or ""` перед `.strip()`.

## СЕРЙОЗНІ ПРОБЛЕМИ (впливають на якість/стабільність)

### ✅ B1. Video processing блокує event loop — ВИПРАВЛЕНО 2026-04-12
**Файли:** `media/video.py:121`, `media/router.py:142,189`
**Суть:** `analyze_video()` — синхронна функція з важким I/O (ffprobe, ffmpeg, API calls). Викликається з async `_build_media_context` БЕЗ `asyncio.to_thread()`. Бот перестає відповідати на будь-які повідомлення поки обробляється відео.
**Порівняння:** `voice.py` правильно загортає в `asyncio.to_thread`.
**Фікс:** `result = await asyncio.to_thread(analyze_video, ...)`.

### ✅ B2. Album photo: відсутній try/except навколо `describe_images` — ВИПРАВЛЕНО 2026-04-12
**Файл:** `media/router.py:136`
**Суть:** Для album photos `describe_images` викликається без обробки виключень. API error крашить весь album handler. Video і audio мають try/except, photo — ні.
**Фікс:** Обгорнути в try/except, повертати порожній опис при помилці.

### ✅ B3. Один поганий album item вбиває весь альбом — ВИПРАВЛЕНО 2026-04-12
**Файл:** `media/router.py:146,157`
**Суть:** Якщо одне відео/аудіо в альбомі не обробилось — `return ("", None)` скидає ВСІ вже зібрані результати інших items. Partial results втрачаються.
**Фікс:** `continue` замість `return` — пропускати битий item, зберігати решту.

### ✅ B4. `vision.py:49` — занадто вузький except — ВИПРАВЛЕНО 2026-04-12
**Файл:** `media/vision.py:49`
**Суть:** `except RuntimeError: return ""` ловить тільки `RuntimeError`. `openai.APIError`, `httpx.TimeoutException`, `ConnectionError` пролітають наскрізь.
**Фікс:** `except Exception`.

### ✅ B5. Telethon: voice = document + voice одночасно — ВИПРАВЛЕНО 2026-04-12
**Файл:** `adapters/userbot.py:43-47`
**Суть:** Voice message встановлює і `has_voice=True` і `has_document=True` (бо в Telethon voice — це document з атрибутами). Може спричинити подвійний routing.
**Фікс:** Виключити voice з `has_document` логіки (аналогічно до video/video_note).

### ✅ B6. Long-term memory: retrieval budget = storage budget (30K токенів) — ВИПРАВЛЕНО 2026-04-12
**Файл:** `memory/manager.py:355-358`
**Суть:** `_select_long_relevant` використовує `_long_budget()` (30K) як ліміт вибірки. Весь long-term може піти в prompt, роздуваючи context window.
**Фікс:** Окремий retrieval budget (наприклад, 6-8K), менший за storage budget.

### ⏭️ B7. Album registry: `threading.Lock` не захищає async flow — NOT A BUG
**Файл:** `media/album_registry.py:30`
**Суть:** `threading.Lock` миттєво проходить у single-thread asyncio. Реальна конкуренція — між coroutines під час `await asyncio.sleep(1.2)`. Lock не захищає claim→settle→process послідовність.
**Фікс:** Використовувати `asyncio.Lock` або переробити на atomic claim pattern.

### ✅ B8. `telegram_bot.py:91` — `filters.ALL` ловить не-message updates — ВИПРАВЛЕНО 2026-04-12
**Файл:** `adapters/telegram_bot.py:91`
**Суть:** Хендлер `filters.ALL` спрацьовує на callback queries, inline queries, poll updates. Фільтрується тільки edited messages.
**Фікс:** Замінити на `filters.TEXT | filters.PHOTO | filters.VIDEO | ...` або додати guard.
